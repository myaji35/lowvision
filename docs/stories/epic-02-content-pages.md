# Epic 2: 사용자 여정 기반 콘텐츠 페이지 구축

## 개요

[진단] → [적응] → [일상] → [권리] → [커뮤니티] 5단계 사용자 여정에 따른 핵심 콘텐츠 페이지를 구축합니다.

## 비즈니스 가치

- 저시력인 가족이 진단 직후부터 일상 적응까지 필요한 정보를 체계적으로 제공
- 정보 탐색 비용 최소화
- 국내 최고의 저시력 정보 허브 포지셔닝

## 사용자 스토리

**As a** 저시력 진단을 받은 가족
**I want** 진단부터 일상 적응까지 단계별 정보를 쉽게 찾고
**So that** 앞으로 무엇을 해야 할지 명확히 알 수 있다

## 수용 기준

- [ ] 5단계 사용자 여정 페이지 모두 구현
- [ ] 각 단계별 최소 3개 이상의 하위 페이지
- [ ] SEO 최적화 (메타 태그, Open Graph)
- [ ] 모바일 반응형 완벽 지원
- [ ] 페이지 로딩 속도 Lighthouse 90점 이상
- [ ] 내부 링크 구조 명확

## 관련 Story

### 1단계: 진단
- Story 2.1: "저시력이란?" 페이지
- Story 2.2: "원인 질환" 페이지 (황반변성, 녹내장 등)
- Story 2.3: "진단 직후 가이드" 페이지

### 2단계: 적응
- Story 2.4: "저시력과 마음 건강" 페이지
- Story 2.5: "가족 및 보호자 지원" 페이지
- Story 2.6: "생애주기별 지원" 페이지

### 3단계: 일상
- Story 2.7: "최신 보조공학(AT) 가이드" 페이지
- Story 2.8: "재활 및 교육" 페이지
- Story 2.9: "디지털 활용 가이드" 페이지

### 4단계: 권리
- Story 2.10: "복지 혜택 총람" 페이지
- Story 2.11: "법률 및 권익 옹호" 페이지
- Story 2.12: "가족을 위한 재정 계획" 페이지

### 5단계: 커뮤니티
