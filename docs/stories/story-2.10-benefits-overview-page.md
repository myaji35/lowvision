## 디자인 요구사항

### 레이아웃
- 상단: "내게 맞는 혜택 찾기" 퀵 가이드 (3-4개 질문으로 필터링)
- 좌측 사이드바: 카테고리 필터 + 고급 필터 (연령대, 등급)
- 메인 영역: 혜택 카드 그리드 (카테고리별 색상 구분)
- 각 카드: 혜택명 + 요약 + "자세히 보기" 버튼
- 상세 페이지: Dialog/Modal 또는 새 페이지

### 접근성
- 시맨틱 HTML (`<aside>` 필터, `<main>` 카드 그리드)
- 필터 체크박스에 명확한 `<label>` 연결
- 검색 입력창에 `role="search"` 적용
- 카드 활성화 시 포커스 트랩 (Modal 사용 시)
- 고대비 모드에서 카테고리 색상 구분 대신 아이콘/텍스트 사용
- 스크린 리더 친화적 ("총 XX개 혜택 중 YY개 표시됨" 안내)

### 데이터 모델
```typescript
interface Benefit {
  id: string;
  name: string;                       // '장애인연금'
  category: BenefitCategory;
  provider: string;                   // '보건복지부'
  summary: string;                    // 1-2줄 요약
  description: string;                // 상세 설명
  eligibility: EligibilityRequirement; // 신청 자격
  benefits: string[];                 // 혜택 내용
  applicationMethod: ApplicationMethod; // 신청 방법
  requiredDocuments: string[];        // 필요 서류
  contact: ContactInfo;
  externalLink?: string;
  deadline?: Date;                    // 신청 기한 (있는 경우)
  updatedAt: Date;                    // 최종 업데이트 날짜
  featured?: boolean;                 // 주요 혜택 여부
}

type BenefitCategory =
  | 'economic'      // 경제 (연금, 수당, 세금 감면)
  | 'health'        // 건강 (의료비, 재활)
  | 'education'     // 교육 (학비, 교육 지원)
  | 'employment'    // 고용 (취업, 창업)
  | 'living'        // 생활 (주거, 에너지, 통신)
  | 'mobility';     // 이동 (교통, 차량)

interface EligibilityRequirement {
  ageRange?: string;                  // '만 18세 이상'
  disabilityGrade?: string[];         // ['1급', '2급', '3급']
  incomeLimit?: string;               // '기준 중위소득 70% 이하'
  residency?: string;                 // '대한민국 국민'
  additionalConditions?: string[];    // 추가 조건
}

interface ApplicationMethod {
  online?: OnlineApplication;
  offline?: OfflineApplication;
  phone?: PhoneApplication;
}

interface OnlineApplication {
  website: string;                    // '복지로 (bokjiro.go.kr)'
  requiresLogin: boolean;
  steps: string[];                    // 신청 절차
}

interface OfflineApplication {
  location: string;                   // '주민센터'
  requiredDocuments: string[];
  processingTime: string;             // '7-14일'
}

interface PhoneApplication {
  phoneNumber: string;
  operatingHours: string;
}

interface ContactInfo {
  phone?: string;
  email?: string;
  website?: string;
  address?: string;
}

interface FilterOptions {
  categories: BenefitCategory[];
  ageRange?: string;
  disabilityGrades?: string[];
  incomeLevel?: string;
}
```

## 주요 기능

### "내게 맞는 혜택 찾기" 퀵 가이드

사용자가 3-4개의 간단한 질문에 답하면 자동으로 필터링된 혜택 목록을 보여줍니다.

**질문 예시:**
1. 연령대를 선택해주세요. (아동/청소년, 청년, 중장년, 노년)
2. 장애 등급을 선택해주세요. (1-3급 중증, 4-6급 경증)
3. 필요한 지원 분야를 선택해주세요. (경제, 건강, 교육, 고용, 생활, 이동)

**결과:**
"총 8개의 혜택이 고객님께 해당됩니다" → 필터링된 카드 표시

## 상호작용 시나리오

### 시나리오 1: 신규 진단자의 혜택 탐색
1. 사용자가 "한눈에 보는 복지 혜택" 페이지 방문
2. "내게 맞는 혜택 찾기" 퀵 가이드 사용 (연령대: 청년, 등급: 중증, 분야: 경제+고용)
3. 필터링된 결과 확인 (장애인연금, 취업 지원 등 8개)
4. "장애인연금" 카드 클릭 → 상세 정보 모달 열기
5. "신청하기" 버튼 클릭 → 복지로 웹사이트 연결 (새 탭)

### 시나리오 2: 특정 혜택 검색
1. 검색창에 "통신비" 입력
2. "통신비 감면" 혜택 카드 1개 표시
3. 카드 클릭하여 상세 정보 확인
4. 이동통신사 문의 전화번호 복사

### 시나리오 3: 카테고리 필터링
1. 좌측 사이드바에서 "경제 지원" 카테고리 체크
2. 경제 관련 혜택만 표시 (장애인연금, 수당, 세금 감면 등)
3. "장애 등급" 필터 추가 (1-3급만 선택)
4. 중증 장애인 대상 경제 혜택만 표시

## 테스트 시나리오

### 접근성 테스트
1. axe DevTools로 자동 검증 (필터, 검색, 카드 구조)
2. 키보드만으로 전체 페이지 탐색 (필터링, 검색, 카드 클릭)
3. NVDA로 필터 변경 시 결과 수 안내 확인
4. 200% 텍스트 확대 시 카드 레이아웃 확인
5. 고대비 모드에서 카테고리 구분 확인

### 기능 테스트
1. 퀵 가이드로 필터링 (3가지 조합 테스트)
2. 검색 기능 (키워드 "연금", "교육", "통신" 등)
3. 카테고리 필터 (단일 선택, 복수 선택)
4. 고급 필터 (연령대, 장애 등급 조합)
5. 카드 클릭 → 상세 정보 모달/페이지 이동
6. 외부 링크 클릭 (새 탭 열기)

### 반응형 테스트
1. 모바일 (320px): 필터가 드롭다운으로 전환, 카드 1열

3. 데스크톱 (1280px+): 좌측 사이드바 필터, 카드 3열
