# 한국저시력인협회 웹사이트

저시력인과 그 가족을 위한 접근성 최우선 웹 플랫폼

## 🎯 프로젝트 개요

이 프로젝트는 한국저시력인협회(KLA)의 웹사이트 리뉴얼 프로젝트로, **WCAG 2.2 Level AA** 접근성 표준을 완벽히 준수하는 현대적인 웹 애플리케이션입니다.

### 핵심 목표
- **기술적 포용성**: 완벽한 웹 접근성 (WCAG 2.2 AA 준수)
- **전략적 유용성**: 사용자 여정 기반 콘텐츠 허브

## 🛠 기술 스택

- **Framework**: Next.js 15+ (App Router)
- **Language**: TypeScript
- **Styling**: TailwindCSS
- **UI Components**: shadcn/ui
- **Database**:
  - 로컬 개발: SQLite (better-sqlite3)
  - 프로덕션: PostgreSQL
- **ORM**: Drizzle ORM

## 🚀 시작하기

### 필수 요구사항

- Node.js 18+
- npm 또는 pnpm

### 설치

\`\`\`bash
# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
\`\`\`

개발 서버가 [http://localhost:3000](http://localhost:3000)에서 실행됩니다.

## ♿ 접근성 (Accessibility)

이 프로젝트는 **WCAG 2.2 Level AA** 표준을 준수합니다:

### 주요 접근성 기능

- ✅ **3가지 테마 모드**: 기본 / 다크 / 고대비 (검정/노랑)
- ✅ **폰트 크기 조절**: 100% / 125% / 150% 확대 지원
- ✅ **키보드 100% 접근성**: 모든 기능을 키보드로 조작 가능
- ✅ **텍스트 200% 확대 지원**: 레이아웃 붕괴 없음 (WCAG 1.4.4)
- ✅ **명확한 포커스 인디케이터**: 2px 두께 고대비 (WCAG 2.4.7, 2.4.13)
- ✅ **포커스 가려짐 방지** (WCAG 2.4.11)
- ✅ **최소 타겟 크기 24x24px** (WCAG 2.5.8)
- ✅ **스크린 리더 지원**: 시맨틱 HTML 및 ARIA
- ✅ **본문 바로가기 링크** (WCAG 2.4.1)

### 접근성 테스트

이 프로젝트는 다층적인 접근성 검증 시스템을 갖추고 있습니다:

#### 1. 개발 중 실시간 검증 (axe-core)

개발 서버 실행 시 자동으로 axe-core가 활성화되어 접근성 위반사항을 콘솔에 표시합니다.

```bash
npm run dev
# 브라우저 콘솔에서 접근성 위반사항 확인
```

#### 2. Lint 검증 (eslint-plugin-jsx-a11y)

코드 작성 시 접근성 규칙을 자동으로 검사합니다.

```bash
npm run lint
```

#### 3. 자동화 테스트 (pa11y)

주요 페이지의 WCAG 2.2 AA 준수 여부를 자동으로 테스트합니다.

```bash
# 개발 서버가 실행 중일 때
npm run test:a11y

# CI/CD 환경에서 (빌드 + 테스트)
npm run test:a11y:ci
```

#### 4. 수동 테스트 체크리스트

- [ ] **키보드 탐색**: Tab/Shift+Tab으로 모든 요소 접근 가능
- [ ] **스크린 리더**: NVDA(Windows) 또는 VoiceOver(macOS)로 전체 탐색
- [ ] **텍스트 확대**: 브라우저에서 200% 확대 시 레이아웃 유지
- [ ] **고대비 모드**: 3가지 테마 모두 명암비 7:1 이상 (AAA)
- [ ] **모바일**: 320px 너비에서 수평 스크롤 없음 (WCAG 1.4.10)
